# Credit Risk Scorecard Pipeline

Credit scoring pipeline built on a Lakehouse Medallion
architecture (Bronze / Silver / Gold).

Raw credit-application CSVs are ingested, validated, cleaned, WoE-encoded,
and reduced to a final feature table that feeds a scikit-learn Logistic
Regression scorecard, tracked in MLflow and served through a Streamlit
dashboard.

## Architecture

```
data/raw/*.csv
      |
      v
  BRONZE   -- GE schema checks --> Parquet (lakehouse/bronze/), untouched
      |
      v
  SILVER   -- DuckDB join + dedup + vintage-analysis TARGET label
           -- scorecardpy WoE binning + IV-based feature selection
           --> Delta tables (lakehouse/silver/silver_clean, silver_woe)
      |
      v
  GOLD     -- WoE features + TARGET only, versioned
           --> Delta table (lakehouse/gold/gold_features)
      |
      v
  MODEL    -- scikit-learn Logistic Regression + scorecard, tracked in MLflow
      |
      v
  STREAMLIT -- portfolio dashboard (passive) + live scoring (active)
```

Every layer has a Great Expectations data-quality checkpoint, and the whole
pipeline is orchestrated as a single Airflow DAG (`lakehouse_pipeline`):
`bronze (3 parallel tasks) -> silver_clean -> silver_woe -> gold_features`.

| Concern | Tool |
|---|---|
| Orchestration | Apache Airflow (standalone mode) |
| Bronze/Silver storage | Parquet |
| Silver/Gold storage | Delta Lake via `deltalake` (delta-rs, no Spark) |
| SQL joins/dedup | DuckDB |
| Data quality | Great Expectations |
| WoE / IV / scorecard | `scorecardpy` |
| Model | scikit-learn Logistic Regression |
| Experiment tracking | MLflow |
| Dashboard | Streamlit |

## Repository structure

```
config/config.py          # single source of truth for storage paths (local vs minio)
ingestion/
  bronze_ingest.py         # Raw CSV -> validated Parquet
  silver_clean.py          # Join, dedup, TARGET label, feature engineering
  silver_woe.py            # WoE binning, IV-based feature selection
  gold_build.py            # Final ML-ready feature table
  dags/lakehouse_dag.py    # Airflow DAG wiring all of the above together
dq_checks/
  ge_utils.py               # shared Great Expectations runner + JSON/HTML report writer
  bronze_suites.py          # Bronze critical/warn expectation suites
  silver_suites.py          # Silver input/output contract suites
  silver_woe_suites.py      # Silver WoE output contract suite
  gold_suites.py            # Gold output contract suite
  reports/                  # generated DQ reports (gitignored, regenerated each run)
docs/
  eda_summary.json, *.png    # EDA notebook outputs (committed)
  iv_report.json             # IV per feature + selected/dropped features (committed)
  woe_plots/                 # WoE bin plots per feature (committed)
  requirements.txt           # pinned Python dependencies
01_EDA_Credit_Risk.ipynb     # exploratory analysis
data/raw/                    # source CSVs (gitignored, see "Getting the data" below)
lakehouse/                   # Bronze/Silver/Gold output (gitignored, regenerated by running the pipeline)
```

## What's not in version control (and why)

| Path | Why it's gitignored | How to get it back |
|---|---|---|
| `venv/` | Python virtual env, machine-specific | `python3 -m venv venv && pip install -r docs/requirements.txt` |
| `data/raw/*.csv` | Kaggle source datasets — large, redistribution restrictions | Download manually (see below), drop into `data/raw/` |
| `lakehouse/` | Bronze/Silver/Gold Parquet+Delta output — fully regenerable | Run the pipeline scripts (see below) |
| `dq_checks/reports/` | Timestamped Great Expectations reports, regenerated every run | Re-run the pipeline; reports are written automatically |
| `mlflow/` | Local MLflow tracking store (model runs, Week 6+) | Regenerated when `ml/train.py` runs |
| `airflow_home/` | Airflow's local metadata DB + generated admin password | Created automatically by `airflow standalone` |
| `.env` | Secrets (e.g. future MinIO credentials) | Never committed; create locally if needed |

what the pipeline produces (EDA plots, IV results, WoE bin plots) is committed under `docs/`.

## Getting started (reproduce locally)

```bash
git clone https://github.com/mehdikai/credit-risk-pipeline.git
cd credit-risk-pipeline

python3 -m venv venv
source venv/bin/activate
pip install -r docs/requirements.txt
```

### Getting the data

Download from Kaggle and place into `data/raw/`:
- "Credit Card Approval Prediction" -> `application_record.csv`, `credit_record.csv`
- "Credit Risk Dataset" -> `credit_risk_dataset.csv`

### Running the pipeline

Either run each layer's script directly, in order:

```bash
python -m ingestion.bronze_ingest
python -m ingestion.silver_clean
python -m ingestion.silver_woe
python -m ingestion.gold_build
```

Or run the whole thing as one Airflow DAG:

```bash
export AIRFLOW_HOME=$(pwd)/airflow_home
export AIRFLOW__CORE__DAGS_FOLDER=$(pwd)/ingestion/dags
airflow standalone
# open http://localhost:8080, trigger "lakehouse_pipeline"
```

## Results so far

- 36,457 clients after Silver join/dedup, **1.69% default rate** (matches
  `docs/eda_summary.json`)
- 16 candidate features evaluated by IV (14 raw + 2 engineered ratios),
  **8 selected** (IV in `[0.02, 0.5]`) — see `docs/iv_report.json`:
  `EMPLOYED_RATIO`, `INCOME_PER_FAM_MEMBER`, `OCCUPATION_TYPE`, `AGE_YEARS`,
  `EMPLOYED_YEARS`, `AMT_INCOME_TOTAL`, `FLAG_OWN_REALTY`, `NAME_FAMILY_STATUS`
- The two engineered ratio features (`EMPLOYED_RATIO` = years employed /
  age, `INCOME_PER_FAM_MEMBER` = income / household size) outperform every
  raw feature on IV while staying interpretable for the scorecard narrative
- `gold_features` (Delta, version 0): 36,457 rows x 8 WoE features + TARGET,
  all Great Expectations checks passing at every layer
- Logistic Regression (`class_weight="balanced"`) trained and tracked in
  MLflow (`mlflow/mlflow.db`, experiment `credit_risk_scorecard`); converted
  into a point-based scorecard (`ml/scorecard.csv`, base 600 pts / 50:1 odds
  / PDO 20). Train/test accuracy (~59%) is only a sanity check here — real
  evaluation (KS/Gini/AUC, threshold selection) is Week 7
